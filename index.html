Google 日曆同步

需要你自己在 Google Cloud Console 建立一組 OAuth 用戶端 ID（Web 應用程式），並將這個網頁實際部署的網址加入「已授權的 JavaScript 來源」，同步才能運作。此功能不需要伺服器，直接在瀏覽器內與 Google 日曆溝通。

尚未連接。

每個專案可各自指定一個日曆。新增／編輯／刪除工作項目時，會自動推送到這裡指定的日曆；按「立即同步」則會把該日曆目前的事件也拉回來比對更新（若日曆上有直接新增的事件，會被匯入為新的工作項目）。

OneDrive 同步

目前所有資料（專案、工作項目、假日、圖層設定）預設存在 Claude 的 artifact 儲存空間，只有在 Claude 裡開啟這個檔案時才讀得到。連接 OneDrive 後，資料會改成存進你自己 OneDrive 帳號下一個給這個 App 專用的隱藏資料夾，手機、桌機只要用同一個微軟帳號登入，打開這個網頁都能讀到同一份、互相同步。

需要你自己在 Microsoft Entra（Azure）後台註冊一個應用程式，取得「應用程式 (用戶端) 識別碼」，並把這個網頁實際部署的網址加進「重新導向 URI」，同步才能運作；不需要伺服器，直接在瀏覽器內與 OneDrive 溝通。為了安全起見，每次重新打開網頁都需要重新登入一次微軟帳號（不會把登入狀態存起來），但已同步的資料本身是永久保存在你的 OneDrive 裡的。

尚未連接，目前資料存在 Claude 的 artifact 儲存空間。
